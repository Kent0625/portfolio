Machine Learning Case Study

The Life of a Bill

A supervised machine learning project that predicts Philippine Senate bill status from legislative records and explains which signals are most useful for understanding bill movement.

Project focus

Full Content

This project turns historical Senate bill records into a classification workflow. The case study highlights the problem, data preparation, model comparison, result interpretation, and value of explaining why many bills remain early in the legislative process.

Problem

Many bills are filed in the Senate, but only a small portion move forward. The project asks whether historical bill attributes can help predict status and reveal common progress patterns.

Data

The dataset contains 7,352 Senate bill records from the 15th to 18th Congress. Records were cleaned, structured, and prepared for modeling with legislative features.

Method

The workflow used Python, feature engineering, model comparison, and XGBoost classification. The project also emphasized explainability through model signals and feature interpretation.

Result

XGBoost was selected as the main model for structured prediction. The output made it easier to compare status signals across filing context, policy area, and legislative movement.

Impact

The project shows how machine learning can support legislative analysis by turning a large public record set into a clearer, evidence-based story about bill progress.
