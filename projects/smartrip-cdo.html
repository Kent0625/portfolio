Project focus

Full Content

This project shows how a zero-cost local tourism prototype can use curated data, deterministic filtering, and prompt engineering to generate itinerary suggestions without relying on expensive external APIs.

Problem

Travel itinerary platforms often require dynamic APIs and commercial data access. The project asks whether a curated local database can still produce useful, grounded itinerary recommendations.

Data

SmartTripCDO uses a manually curated dataset of 47 Cagayan de Oro points of interest, including location context and travel-planning attributes.

Method

The system uses a three-stage static RAG workflow: preference extraction, deterministic filtering with pandas, and grounded itinerary generation through prompt engineering.

Result

The project reported a 0% hallucination rate and 76.32% clustering efficiency, showing that grounded local data can support coherent regional travel planning.

Impact

The prototype gives resource-constrained local tourism teams a practical template for AI-assisted itinerary planning without expensive APIs or complex solvers.
